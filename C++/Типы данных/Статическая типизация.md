Тип проверяется на этапе компиляции. Переменные имеют фиксированный тип, который нельзя изменить.

Это нужно, чтобы:
	- вычислить размер памяти, который будет занимать каждая переменная в программе;
	- определить, какая функция будет вызываться в каждом конкретном месте.

Преимущества статической типизации:
	- проверки типов происходят только один раз - на этапе компиляции;
	- при некоторых дополнительных условиях позволяет обнаруживать потенциальные ошибки уже на этапе компиляции;
	- в интегрированной среде разработки осуществимо более релевантное автодополнение, особенно если типизация - сильная статическая.
