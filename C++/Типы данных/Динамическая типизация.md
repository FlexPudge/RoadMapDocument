Типы проверяются во время выполнения. Переменные могут менять тип в процессе работы программы.
Для этого применяются операторы **dynamic_cast** и **typeid**.



## **RTTI**
**RTTI (Run-Time Type Information)** — механизм в языке C++, который позволяет **определить тип данных переменной или объекта во время выполнения программы**. Это динамическая идентификация типа данных.

RTTI используется в полиморфных сценариях, когда конкретный тип не известен во время компиляции. Например, указатель может указывать на объект базового класса или на объект любого производного класса от базового. 

**Принцип работы**

Стандарт C++ не определяет, как именно реализуется RTTI, и ответственность перекладывается на двоичный интерфейс приложений (ABI). В большинстве компиляторов информация хранится в виртуальной таблице (vtable).

**Ограничение**: RTTI доступен только для полиморфных классов — классов, содержащих хотя бы одну виртуальную функцию. В базовых классах должен быть виртуальный деструктор, чтобы объекты производных классов могли правильно очищаться, если они удаляются по указателю на базовый класс.

## Операторы

Для работы с RTTI в C++ используются два оператора: 

1. **typeid** — возвращает фактический тип объекта, на который указывает указатель (или ссылка). Возвращает ссылку на объект типа type_info, который описывает тип. В классе type_info есть метод name() — возвращает строковое представление типа.
2. **dynamic_cast** — безопасно преобразует указатель (или ссылку) на базовый тип в указатель (или ссылку) на производный тип. Выполняет проверку типа во время выполнения: если объект, на который указывает указатель, не является объектом целевого типа, операция не выполняется, и значение — 0. Если тип — ссылка, выбрасывается исключение типа bad_cast.
